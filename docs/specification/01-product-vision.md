# 🗺️ OfficeCraft AI - 2D 像素数字孪生办公室产品愿景 (Product Vision)

## 一、 核心理念 (Core Concept)
OfficeCraft AI 是一款**面向技术学习者的 2D 像素风虚拟职场数字孪生沙盒**。
我们抛弃了传统的“网页控制台任务列表”，将其重构为一张**可走动、可探索、有空间记忆的 2D 像素虚拟办公室地图**（类似 Gather.town / 星露谷物语风格），并将硬核的技术技能练习、RAG 检索和多智能体博弈无缝编织在空间交互中。

本项目对齐 **赛道三：数字孪生与沉浸交互**。

## 二、 核心愿景 (Core Vision)
- **沉浸式 RPG 空间体验**：打破枯燥单调的表单练习。用户通过键盘 `W-A-S-D` 物理控制像素小人在 25×25 网格的数字孪生办公室中穿梭，让学习技术交付如同探索 RPG 游戏一样生动。
- **真实“团队协同”而非单机解题**：AI 导师不仅是静态的出题或评审机器，更是具有独立人格（Tech Lead 的严厉、数据专家的循循善诱、产品经理的业务驱动）、拥有记忆、会产生观点冲突并共同参与每日晨会的真实团队。
- **空间感知与反馈外化 ("Prompt-to-Light")**：利用前端环境光效感知体系与 8-bit FC 电子音效，将虚拟任务的业务压力（紧急故障）、研习状态（静谧蓝呼吸）及通关成就（金黄欢庆）外化为空间感官体验。
- **物理化场景检索 (Spatial RAG Bookcase)**：将 RAG 知识库与办公室中的“实体书架区”打通。玩家需控制角色走到物理书架前，点击书架触发语义召回，彻底打通物理空间、人、书架与检索上下文。

## 三、 目标用户 (Target Audience)
1. **高校在校生/应届生**：计算机、软件工程、数据科学等相关专业，缺乏真实企业团队协作与复杂研发流程经验，渴望在低成本、高保真的模拟环境中磨炼实战能力。
2. **职业转行与技能精进者**：希望体验真实岗位交付物标准、理解产品与技术架构决策、建立全栈式工程思维，缓解真实求职或工作中的交付焦虑。
3. **极客/复古游戏爱好者**：偏爱 2D 像素风、喜欢高自由度沙盒玩法和 AI 代理协同演进的开发者。

## 四、 空间孪生办公室地图设计 (Spatial Layout)
虚拟办公室采用标准 **25×25 像素网格** 地图（每格 32px），具有碰撞检测，由以下四大核心空间构成：

| 区域名称 | 物理坐标区间 | 包含的交互 NPC/物件 | 空间功能描述 |
| :--- | :--- | :--- | :--- |
| **Lobby (前台大厅)** | `(0,0)` 到 `(8,10)` | 门禁、像素绿植、签到打卡机 | 玩家登录初始落脚点，查看累计总 XP 及成就奖牌墙。 |
| **Dev Bay (研发区)** | `(0,11)` 到 `(12,24)`| 玩家工位、AI 同事工位、像素电脑 | 个人工作站，用户在此进行代码/报告编写，查看本地 Task 看板。任务失败时此区冒出红色报警光晕。 |
| **Meeting Room (会议室)** | `(13,0)` 到 `(24,12)`| 会议圆桌、白板、主管高凌/PM Amy | 发起“每日晨会（站会）”和“多人冲突协调（斡旋博弈）”的剧情场所。 |
| **Archive Room (资料库)** | `(13,13)` 到 `(24,24)`| RAG 实体书架、咖啡机 | 物理化知识库检索地。点击不同书架，语义召回对应 Pandas / 架构设计文档。 |

## 五、 MVP 核心职业与导师阵容
1. **初级软件工程师 (Software Engineer)**  
   - **核心导师**：高凌 (Tech Lead)。严谨、严厉，重视代码质量，反对无底线堆砌功能，口头禅是“先跑一下单测”。  
   - **关卡线**：UserService 的 N+1 查询优化 ➔ 数据库平滑迁移 ➔ 核心高并发架构重构。
2. **初级数据分析师 (Data Analyst)**  
   - **核心导师**：郑莹 (Senior Data Analyst)。专业、循循善诱，善于挖掘数据背后的商业洞察，对新手极其温柔。  
   - **关卡线**：活跃度下降归因分析 ➔ 多维漏斗图转化率调优 ➔ 商业指标预测。
3. **协同配角：Amy (Product Manager)**  
   - **性格**：业务导向、急躁且富有行动力，总想最快上线功能，常与高凌产生架构决策或发布节奏的剧烈冲突，需要用户在会议室进行“多人偏好斡旋调解”。
